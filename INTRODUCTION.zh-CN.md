# Zotero Vault Batch Reading：一个把 Zotero 文献库变成可精读知识库的 Skill

## 它是做什么的

`zotero-vault-batch-reading` 是一个面向论文阅读场景的 Claude/Codex skill。它的目标不是简单“批量总结论文”，而是把 Zotero 里的文献分类，整理成一套结构化、可追溯、可持续扩展的 Markdown 阅读知识库。

它把论文处理分成两层：

- **泛读层**：自动完成机械工作，把一整个 Zotero collection 转成 Markdown，并为每篇论文生成一份可快速浏览的结构化笔记。
- **精读层**：再由 agent 基于全文内容，对真正重要的论文做深入阅读，补出方法、公式、关键结果、原文引用、局限性以及与你研究问题的关联。

这意味着它不是一个“一次性摘要工具”，而是一套从 Zotero 到知识库、从批量筛选到深度阅读的完整工作流。

## 它解决了什么问题

做文献研究时，最常见的几个痛点是：

- Zotero 里论文越来越多，但真正读完、读透的很少。
- 想批量处理文献时，常常只能得到一堆浅层摘要，后面很难继续用。
- PDF、摘要、笔记、研究问题之间是断开的，缺少统一入口。
- 真正有价值的论文，需要回到全文里重新翻方法、数据、公式和原始结论。

这个 skill 的思路是把这些环节接起来：

1. 从 Zotero 直接拉取某个 collection。
2. 自动定位 PDF 并转成 Markdown。
3. 为全部论文生成统一结构的“泛读笔记”。
4. 根据研究主题筛出最值得深入的论文。
5. 对选中的论文生成真正可复用的“精读笔记”。
6. 维护一份总览索引，把原文、泛读、精读关联起来。

## 它的核心特点

### 1. 机械工作和思考工作分开

脚本只负责可靠、重复、机械的部分：

- 读取 Zotero collection
- 找 PDF
- 复制文件
- PDF 转 Markdown
- 写 metadata
- 生成泛读笔记

而真正需要理解论文的部分，例如：

- 这篇文章和当前研究问题有什么关系
- 哪些论文值得精读
- 文中的方法和公式到底怎么理解
- 哪些结果可借鉴，哪些不能直接迁移

这些交给 agent 来做。

这种分工比“让 agent 一口气处理所有事情”更稳，也更适合长期积累知识库。

### 2. 不止是摘要，而是双层阅读结构

它输出的不是单一类型笔记，而是两层结构：

- `03_reading_notes/`：泛读笔记，覆盖 collection 里的全部论文。
- `04_deep_reading_notes/`：精读笔记，只写真正重要的论文。

泛读的作用是**快速扫描和筛选**。  
精读的作用是**沉淀真正能进入研究设计、方法比较和写作引用的内容**。

这样做的好处是：不会为了“批量”而牺牲质量，也不会为了“精读”而失去处理大规模文献的能力。

### 3. 保持全文可追溯

这个 skill 非常强调追溯性。每一篇笔记都会尽量保留：

- Zotero 条目链接
- DOI
- 本地 PDF 路径
- 本地 Markdown 路径
- 原始摘要

因此它不是黑箱式输出。后续无论是回查原文、核对结论，还是在写作时重新确认方法细节，都能快速回到源头。

### 4. 真正围绕研究问题来读论文

它不是“给每篇论文都来一份差不多的总结”，而是先识别你的研究上下文，例如：

- 你研究的是哪类复合灾害或环境风险
- 关注哪个地区
- 更关心暴露、健康负担、机制还是政策
- 重点变量是什么

在此基础上再决定哪些论文值得精读。

因此它特别适合那些已经有明确研究主题、需要围绕某个问题搭建文献脉络的人，而不是单纯做论文搬运。

## 典型使用场景

这个 skill 适合以下场景：

- 你在 Zotero 里已经积累了一整个分类，但还没有系统整理。
- 你想把一批论文转换成 Obsidian / Markdown 可管理的知识库。
- 你需要先批量扫一遍，再挑核心论文做深读。
- 你在写开题、综述、proposal 或论文，需要建立一套与研究问题强相关的文献笔记体系。
- 你不想只要“总结”，而是要可继续用于研究设计和写作的阅读产物。

## 工作流长什么样

这套 skill 的工作流可以概括为五步：

1. **Onboarding**
   先明确研究问题和阅读策略。

2. **Prepare**
   从 Zotero collection 中读取条目、复制 PDF、转 Markdown。

3. **Skim**
   为所有论文生成泛读笔记，形成第一层知识库。

4. **Recommend + Deep Reading**
   根据研究主题筛论文，再对核心论文写精读笔记。

5. **Overview**
   更新总览索引，把 collection、泛读、精读连起来。

它的最终输出通常会包含：

- `01_original_pdf/`
- `02_original_md/`
- `03_reading_notes/`
- `04_deep_reading_notes/`
- `00_collection_overview.md`
- `_ProcessLog_进度记录.md`

## 它适合谁

如果你符合下面任一情况，这个 skill 会比较有价值：

- 你做科研，需要长期读论文而不是一次性读几篇。
- 你已经在用 Zotero 管理文献。
- 你希望把文献阅读和知识库建设合并成一个流程。
- 你希望保留“批量处理效率”，同时又不放弃“精读质量”。
- 你需要让文献阅读结果能够直接服务于研究设计、方法借鉴和论文写作。

## 它不追求什么

这个 skill 不是为了生成表面上很漂亮、但后续用不上的“万能摘要”。  
它更重视三件事：

- **结构化**
- **可追溯**
- **服务研究问题**

所以它输出的内容可能没有那种一眼看上去很“华丽”的总结感，但更适合真实研究工作流。

## 一句话总结

如果说普通论文总结工具是在“帮你看过论文”，那么 `zotero-vault-batch-reading` 更像是在**帮你把 Zotero 里的论文，整理成一个能继续思考、比较、精读和写作的研究型阅读系统**。
